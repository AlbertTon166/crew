## 📊 任务调度执行示例

### 单轮执行流程（协作轮次 ≤ 3）

```
Step 1: Manager 分配任务
  → Worker-1: 异常处理（子任务 1/3）
  → Worker-2: API 网关（子任务 1/3）

Step 2: Worker 报告进度（工具调用 1-2 次）
  → Manager 评估是否继续或调整

Step 3: 最终确认或终止
  → 通过 → 进入下一任务
  → 失败 2 次 → 终止并记录
```

### 超时控制

| 任务大小 | 最大步数 | 超时时间 | 处理方式 |
|----------|----------|----------|----------|
| 简单（≤30min） | 10 步 | 35min | 超时终止，降级处理 |
| 中等（≤60min） | 15 步 | 70min | 超时终止，降级处理 |
| 复杂（≤120min） | 20 步 | 140min | 超时分段，保留中间状态 |
| 超复杂（≤180min） | 25 步 | 200min | 超时分段，人工介入 |

## ⏰ 每日工作时间线（北京时间 UTC+8）

```
08:00 北京时间  → Planner 生成当日任务列表（基于并行组）
09:00 北京时间  → Sprint 任务执行（并行组 A/B/C 轮转）
10:00 北京时间  → 中间 check（超时评估）
12:00 北京时间  → Sprint 任务继续 / 串行任务推进
14:00 北京时间  → Reviewer 代码审查
15:00 北京时间  → Tester 测试验证
16:00 北京时间  → Deployer 部署（如通过）
17:00 北京时间  → 人工 check（马楼）← 例行检查点
20:00 北京时间  → 晚间 check + 次日规划
07:00 北京时间（次日）  → 日志归档 + Sprint 总结
```
